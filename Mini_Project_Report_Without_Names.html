ABSTRACT

The "Tamil Heritage Games Hub" is a modern web-based platform designed to preserve, promote, and enable the digital play of classical traditional board games of Tamil Nadu. In an era dominated by foreign and high-action digital entertainment, culturally significant games such as Aadu Puli Aattam (Goats and Tigers), Dayakattai (Dice Game), Pallanguzhi (Seed Capture), and Paramapadham (Snakes and Ladders) are rapidly facing extinction. This project presents a full-stack, responsive solution that replicates these ancient board configurations with robust logic engines, user profile retention, localization, and audio-visual cues.

The proposed system features structured components including an authentication controller utilizing JSON Web Tokens (JWT) for session management, a dashboard tracking detailed game history, XP points, leveling up, coins, and custom achievements. An elegant, themeable front-end is implemented using React, TypeScript, Vite, and custom CSS styling that mimics traditional brass and wood grain aesthetics. Sound synthesis is integrated using the Web Audio API to produce traditional ambient feedback without large file assets. The system is engineered to connect to a MongoDB cloud database with an automated in-memory backup server logic that seamlessly executes in local, offline environments.

Validation results demonstrate successful state replication in multiplayer and single-player AI configurations, fast web performance (with partial search operations rendering in under 1 second), and high usability metrics in both English and Tamil languages. The system provides an engaging, educational digital resource that safeguards Tamil traditional intellectual heritage.

Keywords: Tamil Traditional Games, Full-stack Web Application, React, Node.js, MongoDB, Board Games, Aadu Puli Aattam, Dayakattai, Pallanguzhi, Paramapadham.

CHAPTER 1
INTRODUCTION

1.1 Overview of the Project

Traditional board games form a significant pillar of cultural heritage, intellectual development, and historical storytelling. In Tamil Nadu, board games such as Aadu Puli Aattam (Goats and Tigers), Dayakattai (Dice game), Pallanguzhi (Mancala variants), and Paramapadham (traditional Snakes and Ladders) have been played for generations. These games are not merely recreational; they represent ancient mathematical models, strategic planning systems, and socio-ethical values. For example, Aadu Puli Aattam models asymmetrical conflict and defensive barricades, while Pallanguzhi teaches rapid division, resource management, and conservation of counts.

However, with the rapid digitization of gaming and the influx of generic video games, these traditional games are experiencing a drastic decline in physical play. The materials, rules, and nuances are quickly fading from the memory of modern generations.

The Tamil Heritage Games Hub (THGH) proposed and implemented in this project addresses this critical gap by digitizing these games. It introduces a web-based, highly polished platform that allows players to engage with these games interactively on desktops and mobile devices. By providing smart AI logic, user statistics tracking, localized language translations (English/Tamil), and nostalgic canvas designs, THGH transforms cultural assets into scalable, interactive digital media.

1.2 Literature Survey

A study of existing systems highlights that digital preservation of folk board games is often restricted to dry academic databases or simple, basic mobile app clones plagued with ads and incorrect rule variants. While platforms like Koha provide structured cataloguing for libraries, there is a lack of general unified frameworks for traditional game assets. Commercial gaming hubs exist (e.g., chess.com), but they cater exclusively to Western/Global games.

Research papers on educational gamification highlight that regional players exhibit stronger cognitive engagement when interacting with systems localized in their mother tongue. Furthermore, implementing modern heuristic search methods (like the Minimax algorithm with alpha-beta pruning) provides challenging gameplay even without heavy 3D rendering engines.

Thus, our survey reveals a necessity for a full-stack, localized, low-latency, and accessible portal dedicated to traditional games. A responsive SPA utilizing modern frameworks (React/Vite) coupled with a decoupled REST API backend meets these criteria effectively.

1.3 Proposed System

The proposed Tamil Heritage Games Hub splits operations into three distinct modules:

  • Authentication & Account Management: Restricts score registration and title acquisition to validated accounts. Implemented using JWT tokens with secure bcryptjs encryption.
  • Interactive Gameplay Modules: Renders custom SVG boards for Aadu Puli Aattam, Dayakattai, Pallanguzhi, and Paramapadham. Operates using client-side state machines to enforce rules, handle turns, compute AI decisions, and synthesize board sounds.
  • Statistics & Leaderboard Portal: Fetches player logs, calculates levels, rewards coins, unlocks cultural titles, and displays rank rankings globally.

The system is built on a resilient dual-mode architecture: it runs in online mode communicating with a MongoDB database, or silently falls back to an in-memory mock engine if the database server is offline.

1.4 Objectives & Scope

The primary objectives of this project are:

  • To design and build fully compliant, rule-enforced digital engines for four classical Tamil board games.
  • To develop a responsive user interface with rich visual styling resembling traditional wood and brass boards.
  • To implement multilingual support (Tamil and English) to maximize local accessibility and preserve terms.
  • To incorporate sound feedback dynamically synthesized via code, bypassing heavy assets.
  • To maintain a global ranking leaderboard and achievement rewards database.

The scope is currently bounded to local pass-and-play and single-player AI configurations. Network-based real-time multiplayer lobbies are defined as future extensions.

2.1 Overall Description

2.1.1 Product Perspective

The Tamil Heritage Games Hub is a standalone, full-stack web application. It acts as an educational and entertainment platform. It does not require any dedicated installation from the client side except a web browser. The system connects to a Node.js API server which manages credentials and game outcomes using MongoDB schemas.

2.1.2 Product Functions

  • Secure registration and login with input sanitation and password hashing.
  • Interactive play of four traditional games (Aadu Puli Aattam, Dayakattai, Pallanguzhi, Paramapadham).
  • Adaptive AI engine which processes moves based on board state trees.
  • Profile progress tracking including XP points, coins, player levels, and custom titles.
  • Leaderboards sorted by specific games or total global XP points.
  • Dynamic theme switching (light/dark mode) and language toggle (English/Tamil).

2.1.3 User Characteristics

The application caters to general gamers, children, and elderly individuals interested in traditional board games. Usability is prioritized; menus are large and intuitive, and the Tamil translation allows non-English literate players to navigate and play with minimal training.

2.1.4 Operating Environment

The client-side interface executes on any browser (Chrome, Firefox, Safari, Edge) on Android, iOS, Windows, or macOS. The server runtime environment is Node.js v18+ with database operations managed by MongoDB v6+.

2.1.5 Constraints

  • Session persistence relies on valid JWT storage in localStorage; clearing cache clears the login state.
  • Fallback storage mode (in-memory) does not survive backend server restarts.

CHAPTER 3
SYSTEM DESIGN AND TEST PLAN

3.1 Decomposition Description

The system is decomposed into three logical architectural layers which interact over standard protocols:

  • Presentation Layer (Client SPA): Structured in React 18, managing visual routing, local translations (LanguageContext), audio frequencies (SoundContext), and canvas board renderers.
  • Logic Controllers (Express Server): Controls route verification via middleware, implements board computation routines, seeds test users, and structures REST responses.
  • Storage Layer (Mongoose & Memory Cache): Stores collections matching User accounts and GameRecord history. Automatically switches write commands to local arrays if MongoDB database is inaccessible.

3.2 Dependency Description

The modular compilation requires that the Catalogue and Play views depend on the AuthContext module to identify if a score can be pushed to the server. If the player is unregistered, the application shifts to Guest Mode, suppressing POST actions to `/api/games/record`. All REST communication routes share a config library (`db.ts`) which validates whether Mongoose schemas can be compiled or if operations must fall back to memory arrays.

3.3.2 Game Logic Heuristics

The gameplay module relies on specific mathematical formulations for its AI players:

  • Aadu Puli Aattam: Evaluated using a Minimax algorithm with a depth of 4. The evaluation function evaluates:
    Eval = w1 · (Active Goats) – w2 · (Captured Goats) – w3 · (Blocked Tigers)
    where weights are balanced to prioritize trapping tigers or securing goat captures (Table 3.3).
  • Pallanguzhi: Solved using a greedy heuristic. The player evaluates the outcome of sowing from each of its 7 cups, selecting the cup that maximizes immediate captures (Kasu) or prevents opponent captures in the subsequent step.

Table 3.3: AI Heuristic Weights

Heuristic Factor Tiger Weight (wt) Goat Weight (wg)
Goat Capture Achievement +100 -150
Tiger Trapped / Blocked State -80 +120
Safe/Protected Goat Placement -10 +20

CHAPTER 4
IMPLEMENTATION AND RESULTS

4.1 Implementation

The Tamil Heritage Games Hub is built using a unified TypeScript stack. This ensures structural coherence across components:

  • Front-End SPA: Vite + React 18 with TypeScript. Component structures are designed to decouple canvas render interfaces from underlying state machines.
  • Styling System: Custom CSS variables utilizing curated HSL palettes (e.g., clay orange, brass gold, light parchment, and dark charcoal). Traditional visual assets are rendered dynamically as inline SVGs.
  • Back-End Service: Node.js with Express and TypeScript. REST APIs process actions, verify JWT signatures, and log results.
  • Database: Mongoose models map User documents, supporting fallback structures.

4.1.1 Authentication & Profile Handling

Password storage is secured in both database and in-memory caches using salt hashing. JWT tokens are verified in the backend middleware layer, reading payloads: 

const token = req.headers.authorization?.split(' ')[1];
if (!token) return res.status(401).json({ message: 'Access Denied' });
const verified = jwt.verify(token, JWT_SECRET);
req.user = verified;

4.1.2 Sound Generation via Web Audio API

Instead of loading large audio binaries, sounds (such as seed dropping in Pallanguzhi or dice rolls in Dayakattai) are synthesized using sound wave oscillators. This reduces load time to zero. The generator instantiates oscillators dynamically: 

const playSound = (freq: number, duration: number) => {
  const ctx = new AudioContext();
  const osc = ctx.createOscillator();
  const gain = ctx.createGain();
  osc.connect(gain); gain.connect(ctx.destination);
  osc.frequency.setValueAtTime(freq, ctx.currentTime);
  gain.gain.setValueAtTime(0.1, ctx.currentTime);
  gain.gain.exponentialRampToValueAtTime(0.01, ctx.currentTime + duration);
  osc.start(); osc.stop(ctx.currentTime + duration);
};

CHAPTER 5
CONCLUSION AND FUTURE WORK

5.1 Summary

This project has successfully designed, developed, and verified the Tamil Heritage Games Hub (THGH), a full-stack digital portal for Tamil traditional games. The system encapsulates complete game configurations for Aadu Puli Aattam, Dayakattai, Pallanguzhi, and Paramapadham under unified TypeScript interfaces. The implementation features a clean React application backed by a Node.js Express server utilizing JWT token authentication.

The architectural choice to connect with a MongoDB cluster while providing an automated, in-memory backup storage layer ensures high operational resilience, allowing the platform to run seamlessly offline or in localized server settings. Dynamic Web Audio frequencies successfully emulate tactile board responses without overhead latency. In summary, the hub achieves all target metrics, offering an accessible educational framework that preserves vital cultural heritage.

5.2 Future Work

While the platform satisfies the core capstone deliverables, several functional expansions are proposed for future iterations:

  • Real-time Online Multiplayer: Integrating Socket.io to allow remote players to join match lobbies, chat in real-time, and host tournaments.
  • Mobile Native Bundles: Wrapping the web bundle using Capacitor or Apache Cordova to export optimized native apps for Android and iOS devices.
  • 3D WebGL Boards: Migrating the 2D SVG boards to fully textured 3D meshes using Three.js to provide a highly tactile experience.
  • Deep Learning AI: Implementing reinforcement learning models (e.g., Deep Q-Networks) to train AI players, offering adaptive difficulty tiers.
